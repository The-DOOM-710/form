STEP 01

Add the Script Tag

Paste this tag just before your closing </body>. That's your entire setup ‚Äî the widget handles the rest of the pipeline handoff internally.

HTML
<script src="https://cdn.jsdelivr.net/gh/The-DOOM-710/form/form.js"></script>

‚ö†Ô∏è Do not use type="module" or import this inside a bundler ‚Äî it must load as a global script.

// step 02
STEP 02

React / Vite

Create a thin wrapper component once and reuse it across your entire app. The widget mounts into the DOM ref and manages its own lifecycle.

src/components/AcmeForm.tsx
TypeScript / React
import { useEffect, useRef } from "react"

declare global {
  interface Window {
    AcmeFormWidget: {
      mountForm: (
        el: HTMLElement,
        formType: string,
        style?: any
      ) => void
    }
  }
}

export default function AcmeForm({
  formType, style
}: { formType: string; style?: any }) {
  const ref = useRef(null)

  useEffect(() => {
    if (!ref.current) return
    window.AcmeFormWidget?.mountForm(ref.current, formType, style)
  }, [formType])

  return <div ref={ref} />
}

Drop it anywhere in your JSX ‚Äî swap formType to render any of the 13 form types:

JSX
<AcmeForm formType="nps" />
<AcmeForm formType="supportTicket" />
<AcmeForm formType="churnSurvey" />

STEP 03

Next.js (App Router)

Load the script once in your root layout using the afterInteractive strategy ‚Äî then use the same AcmeForm component across any page or route.

app/layout.tsx
TypeScript / Next.js
import Script from "next/script"

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Script
          src="https://cdn.jsdelivr.net/gh/The-DOOM-710/form/form.js"
          strategy="afterInteractive"
        />
      </body>
    </html>
  )
}

Then use the component inside any page:

JSX
<AcmeForm formType="betaFeedback" />

STEP 04

Plain HTML

No framework required. Load the script and call mountForm directly ‚Äî the response still flows through the full pipeline.

HTML
<!-- 1. Mount target -->
<div id="form-root"></div>

<!-- 2. Widget -->
<script src="https://cdn.jsdelivr.net/gh/The-DOOM-710/form/form.js"></script>

<!-- 3. Mount -->
<script>
  AcmeFormWidget.mountForm(
    document.getElementById("form-root"),
    "productFeedback"
  )
</script>

NOTES

Things to Know

‚ö°
Script load order matters. Always ensure form.js has fully loaded before calling mountForm. Use strategy="afterInteractive" in Next.js, or place your init script after the widget tag in vanilla HTML.
üîÅ
Switch forms at runtime. Update the formType prop dynamically to render different forms in response to user actions or lifecycle events ‚Äî no page reload required.
üì¶
Payload enrichment. Certain forms expect additional context in the submission payload ‚Äî for example, ticket_id for CSAT, deal_id for Win/Loss, and feature_id for Beta Feedback. The Node.js preprocessing layer uses these to route and enrich data before AI analysis.
ü§ñ
AI agents handle the heavy lifting. Once responses reach the backend, AI agents classify intent, normalise free-text, cluster themes across responses, detect sentiment drift, and generate priority-ranked insight summaries ‚Äî all without manual tagging.

REFERENCE

13 Available Form Types

Pass any of these strings as the formType value. Each form is schema-mapped to the preprocessing pipeline ‚Äî fields, validation, scoring logic, and AI routing are all pre-configured. You just trigger the right one at the right moment.

What the pipeline receives
Typed, structured payloads ‚Äî not raw form data
Every form submission is validated against its schema before leaving the widget. The Node.js layer receives clean, typed data it can immediately normalise, segment, and route ‚Äî no scrubbing required.
What the AI agents do with it
Classify, cluster, score, and summarise at scale
AI agents apply form-specific logic ‚Äî NPS promoter/passive/detractor segmentation, CSAT dimension weighting, churn reason clustering, feature request prioritisation ‚Äî and surface the signal that actually matters.
